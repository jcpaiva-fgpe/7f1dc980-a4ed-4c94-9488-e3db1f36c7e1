        <ul> 
         <li><font style="vertical-align: inherit;"><font style="vertical-align: inherit;">declare the properties of the class as </font></font><code>private</code><font style="vertical-align: inherit;"><font style="vertical-align: inherit;">.</font></font></li> 
         <li><font style="vertical-align: inherit;"></font><code>public</code> <strong><em><font style="vertical-align: inherit;"><font style="vertical-align: inherit;">develop get</font></font></em></strong><font style="vertical-align: inherit;"><font style="vertical-align: inherit;"> and </font></font><strong><em><font style="vertical-align: inherit;"><font style="vertical-align: inherit;">set</font></font></em></strong><font style="vertical-align: inherit;"><font style="vertical-align: inherit;"> methods </font><font style="vertical-align: inherit;">to access and update the value of each declared property </font></font><code>private</code><font style="vertical-align: inherit;"><font style="vertical-align: inherit;">.</font></font></li> 
        </ul> 
        <h3><font style="vertical-align: inherit;"><font style="vertical-align: inherit;">Advantage</font></font></h3> 
        <ul> 
         <li><font style="vertical-align: inherit;"><font style="vertical-align: inherit;">Better control of properties and methods.</font></font></li> 
         <li><font style="vertical-align: inherit;"><font style="vertical-align: inherit;">Flexibility when it comes to modifying one part of the code without affecting other parts.</font></font></li> 
         <li><font style="vertical-align: inherit;"><font style="vertical-align: inherit;">Increases data security.</font></font></li> 
        </ul> 
        <h3><font style="vertical-align: inherit;"><font style="vertical-align: inherit;">Get and Set</font></font></h3> 
        <p><font style="vertical-align: inherit;"><font style="vertical-align: inherit;">We learned that properties </font></font><code>private</code><font style="vertical-align: inherit;"><font style="vertical-align: inherit;">can only be accessed within the same class. </font><font style="vertical-align: inherit;">However, they can be accessed through </font></font><em><strong><font style="vertical-align: inherit;"><font style="vertical-align: inherit;">get</font></font></strong></em><font style="vertical-align: inherit;"><font style="vertical-align: inherit;"> and </font></font><strong><em><font style="vertical-align: inherit;"><font style="vertical-align: inherit;">set</font></font></em></strong><font style="vertical-align: inherit;"><font style="vertical-align: inherit;"> methods declared as </font></font><code>public</code><font style="vertical-align: inherit;"><font style="vertical-align: inherit;">.</font></font></p> 
        <p><font style="vertical-align: inherit;"><font style="vertical-align: inherit;">A method </font></font><code>get</code><font style="vertical-align: inherit;"><font style="vertical-align: inherit;">will return the value of the variable, and the method will </font></font><code>set</code><font style="vertical-align: inherit;"><font style="vertical-align: inherit;">modify its value.</font></font></p> 
        <p><font style="vertical-align: inherit;"><font style="vertical-align: inherit;">The syntax for both methods is the same: they start with </font></font><code>get</code><font style="vertical-align: inherit;"><font style="vertical-align: inherit;">or </font></font><code>set</code><font style="vertical-align: inherit;"><font style="vertical-align: inherit;">followed by the property name with the first letter capitalized.</font></font></p> 
        <h3><font style="vertical-align: inherit;"><font style="vertical-align: inherit;">Example:</font></font></h3> 
        <blockquote> 
         <p><code>public class OOP_ejemplos {</code></p> 
         <p><code>&nbsp;&nbsp;&nbsp; public static void main(String[] args) {<br> &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; SubClase miObj = new SubClase(4, 3);<br> &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; System.out.println(miObj.suma());<br> &nbsp;&nbsp;&nbsp; }</code></p> 
         <p><code>}</code></p> 
         <p><code>class SubClase {</code></p> 
         <p><code>&nbsp;&nbsp;&nbsp; private int x, y;<br> &nbsp;&nbsp; &nbsp;<br> &nbsp;&nbsp;&nbsp; public SubClase(int _x, int _y) {<br> &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; setX(_x);<br> &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; setY(_y);<br> &nbsp;&nbsp;&nbsp; }</code></p> 
         <p><code>&nbsp;&nbsp;&nbsp; public int getX() {<br> &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; return x;<br> &nbsp;&nbsp;&nbsp; }</code></p> 
         <p><code>&nbsp;&nbsp;&nbsp; public int getY() {<br> &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; return y;<br> &nbsp;&nbsp;&nbsp; }</code></p> 
         <p><code>&nbsp;&nbsp;&nbsp; public void setX(int x) {<br> &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; this.x = x;<br> &nbsp;&nbsp;&nbsp; }</code></p> 
         <p><code>&nbsp;&nbsp;&nbsp; public void setY(int y) {<br> &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; this.y = y;<br> &nbsp;&nbsp;&nbsp; }</code></p> 
         <p><code>&nbsp;&nbsp;&nbsp; public int suma() {<br> &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; return getX() + getY();<br> &nbsp;&nbsp;&nbsp; }<br> }</code></p> 
        </blockquote> 
        <p><font style="vertical-align: inherit;"><font style="vertical-align: inherit;">You can see one method </font></font><code>get</code><font style="vertical-align: inherit;"><font style="vertical-align: inherit;">and another method </font></font><code>set</code><font style="vertical-align: inherit;"><font style="vertical-align: inherit;">for each of the properties.</font></font></p> 
        <p><font style="vertical-align: inherit;"><font style="vertical-align: inherit;">The this keyword is used to refer to the object with which the method was invoked.</font></font></p>
